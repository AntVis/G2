# Controller

> Controller 概念应该来源于久远的 MVC 开发模式吧。简单理解 C 就是两层（M 数据，V 视图）之间的衔接！

G2 这里的 Controller 主要作用：通过获取 View 中的数据、配置，然后生成和管理对应的实例（视图组件、逻辑组件）。

主要分成为三类：

- 组件
- 事件
- 坐标系

## 组件

提供：

1. 组件生成、渲染、布局、管理
2. API

## 事件

提供：

1. plot 生命周期事件
2. 移动端手势事件 gesture
3. 冒泡机制

## 坐标系

提供：

1. 坐标系生成，管理
2. 坐标系的 API 透出
